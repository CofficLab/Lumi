## 根因

重构期间 `LumiCoreKit` 里持有 keychain `service = "com.coffic.lumi.apikey"` 的 `LumiAPIKeyStore` 被删除，所有 provider 改用 `KeychainStore.shared`，而后者是 `KeychainStore(service: "")`（空 service，注释写着 "must be configured before use" 却从未配置）。

keychain 按 `(service, account)` 寻址：旧 key 存在 `(service: "com.coffic.lumi.apikey", account: "DevAssistant_ApiKey_StepFun")`，新代码去 `(service: "", account: ...)` 读，读不到 → 提示输入 key。已用 `security` 命令实锤：旧 service 下有 14 个 provider 的 key（StepFun 在内），空 service 下一个都没有。

**影响面**：所有远程 provider，不止 StepFun。

## 修复方案（已确认：恢复 Lumi 专属 store 单例）

KeychainKit 保持通用（`shared` 留空不动），在 `LumiLLMProviderSupport` 里恢复一个 Lumi 专属 keychain store 单例，service = `"com.coffic.lumi.apikey"`，让所有 provider 走它。旧 key 原地可用，无需迁移。

### 改动清单

**1. 新增** `Packages/LumiLLMProviderSupport/Sources/Utilities/LumiAPIKeyStore.swift`
   - 一个 `public final class LumiAPIKeyStore`，内部持有 `KeychainStore(service: "com.coffic.lumi.apikey")`。
   - 暴露 `shared` 单例，以及 `string(forKey:)` / `set(_:forKey:)` / `remove(forKey:)` / `loadMigratingLegacyUserDefaults(forKey:)`（保留 UserDefaults→Keychain 迁移语义，复用 `KeychainStore` 实现）。
   - 命名沿用历史名 `LumiAPIKeyStore`，降低认知成本，也让残留的注释/文档重新对上。

**2. 改** `Packages/LumiLLMProviderSupport/Sources/Utilities/LumiAPIKeyTools.swift`
   - 把 5 处 `KeychainStore.shared` 换成 `LumiAPIKeyStore.shared`。这一处覆盖了 ~25 个 provider 和所有走 `LumiAPIKeyTools` 的 renderer/service。

**3. 改 3 个直接 bypass `LumiAPIKeyTools` 的调用点**（保持全局一致，都用同一个 service）：
   - `Plugins/LLMProviderMiniMaxPlugin/Sources/Tools/MiniMaxVideoTool.swift:41` — `KeychainStore.shared` → `LumiAPIKeyStore.shared`（这是真正读 key 的，必须改）。
   - `Plugins/LLMProviderCodexPlugin/Sources/CodexProvider.swift:24` 和 `Plugins/LLMProviderMLXPlugin/Sources/MLXProvider.swift:49` — 这俩 `apiKeyStorageKey == ""`，`loadMigratingLegacyUserDefaults(forKey: "")` 本就 guard 返回 nil，改成 `LumiAPIKeyStore.shared` 行为不变但保持一致（顺带移除多余的 `import KeychainKit`，改 import `LumiLLMProviderSupport`，与文件其余部分一致）。

**4. 顺手清理**（非必需，保持文档准确）：
   - `Plugins/LLMProviderMiniMaxPlugin/Sources/Tools/MiniMaxVideoModels.swift:166` 的过时注释。

### 不改的地方
- `KeychainKit/Sources/KeychainStore.swift`（通用包，shared 留空）。
- 设置页 `RemoteProviderSettingsPage.swift` / `ProviderAPIKeyInputView.swift`（只走 `LumiLLMProvider` 协议，传递修复）。
- ~25 个 provider 文件（都调 `LumiAPIKeyTools`，传递修复）。
- Xiaomi renderer 里硬编码的 storage-key 字符串（account 没变，不受 service 改动影响）。

### 验证
- `xcodebuild -scheme Lumi -configuration Debug -destination 'platform=macOS' build` 通过。
- 运行 App，StepFun 不再提示输入 key（旧 key 在旧 service 下可读到）。
