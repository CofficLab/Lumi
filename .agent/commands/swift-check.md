# Swift 代码检查向导

帮助用户检查 Swift 代码质量和规范遵循情况。

## 检查项目

### 1. 代码规范

- [ ] 命名规范（PascalCase, camelCase）
- [ ] 代码组织结构
- [ ] MARK 注释使用
- [ ] 访问控制修饰符

### 2. 代码质量

- [ ] 函数长度（不超过 50 行）
- [ ] 类型复杂度
- [ ] 重复代码
- [ ] 未使用代码

### 3. Swift 特性

- [ ] 使用最新 Swift 语法
- [ ] 正确使用 async/await
- [ ] 正确使用 Result 类型
- [ ] 错误处理完善

### 4. 内存管理

- [ ] 循环引用检查
- [ ] weak/unowned 使用
- [ ] 闭包捕获列表

### 5. 性能

- [ ] 避免不必要的计算
- [ ] 正确使用缓存
- [ ] 延迟加载

## 工作流程

### 1. 静态分析

```bash
# 使用 SwiftLint（如果已安装）
swiftlint lint
```

### 2. 代码审查

逐项检查上述清单

### 3. 生成报告

```markdown
# 代码检查报告

## 问题汇总

- 严重：X 个
- 警告：Y 个
- 建议：Z 个

## 详细问题

### 严重问题

1. 问题描述
   - 文件：xxx.swift
   - 行号：123
   - 修复建议：...

## 总体评价

...
```

### 4. 提供修复建议

对每个问题提供：
- 问题说明
- 修复方案
- 代码示例

## 规则

- 优先修复严重问题
- 提供具体修复代码
- 解释问题原因
- 用中文报告
