# 重构和清理向导

帮助用户重构代码并保持代码整洁。

## 工作流程

### 1. 代码分析

- 识别代码异味
- 查找重复代码
- 分析复杂度

### 2. 制定重构计划

确定重构目标：
- 提高可读性
- 减少重复
- 改善结构
- 优化性能

### 3. 执行重构

遵循步骤：
1. 确保测试通过
2. 小步重构
3. 每步验证
4. 提交变更

### 4. 清理工作

- 删除死代码
- 更新注释
- 整理导入

## 重构原则

### SOLID 原则

- **S**ingle Responsibility - 单一职责
- **O**pen/Closed - 开闭原则
- **L**iskov Substitution - 里氏替换
- **I**nterface Segregation - 接口隔离
- **D**ependency Inversion - 依赖倒置

### DRY 原则

Don't Repeat Yourself - 避免重复

### KISS 原则

Keep It Simple, Stupid - 保持简单

## 常见重构模式

- 提取方法
- 重命名变量/函数
- 合并重复条件
- 拆分大函数
- 移除未使用代码

## 规则

- 重构不改变行为
- 小步迭代
- 频繁提交
- 用中文交流
