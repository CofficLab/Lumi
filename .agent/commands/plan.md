# 项目规划向导

帮助用户规划项目或功能的开发。

## 工作流程

### 1. 需求分析

- 明确目标
- 确定范围
- 识别约束条件

### 2. 技术选型

- 评估现有组件
- 选择合适的技术方案
- 考虑扩展性

### 3. 任务分解

将大目标分解为可执行的小任务：

```
- [ ] 任务 1
  - [ ] 子任务 1.1
  - [ ] 子任务 1.2
- [ ] 任务 2
```

### 4. 风险评估

识别潜在风险：
- 技术难点
- 依赖问题
- 时间限制

### 5. 输出计划

提供清晰的开发计划文档

## 输出格式

```markdown
# 项目名称

## 目标

简要描述...

## 范围

### 包含
- 功能 1
- 功能 2

### 不包含
- ...

## 技术方案

...

## 任务列表

- [ ] ...

## 风险与对策

...

## 预估时间

...
```

## 规则

- 保持计划简洁可行
- 优先实现核心功能
- 预留缓冲时间
- 用中文编写
