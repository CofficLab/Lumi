## 重构流程

### 1. 代码分析

#### 识别代码异味

| 异味 | 说明 | 严重性 |
|------|------|--------|
| 长函数 | 函数超过 50 行 | 🔴 |
| 大类 | 类超过 500 行 | 🔴 |
| 重复代码 | 相同代码多次出现 | 🔴 |
| 过长参数列表 | 参数超过 5 个 | 🟡 |
| 数据泥团 | 经常一起出现的数据 | 🟡 |
| 过度耦合 | 模块间依赖过多 | 🔴 |
| 未使用代码 | 死代码 | 🟢 |

#### 复杂度分析

```bash
# 使用 SwiftLint 检查复杂度
swiftlint rules | grep complexity
```

### 2. 制定重构计划

#### 优先级排序

| 优先级 | 标准 | 示例 |
|--------|------|------|
| P0 | 影响稳定性 | 内存泄漏、崩溃 |
| P1 | 影响可维护性 | 重复代码、复杂函数 |
| P2 | 影响可读性 | 命名不清、缺少注释 |
| P3 | 优化改进 | 性能优化、代码简化 |

#### 风险评估

- 影响范围
- 测试覆盖率
- 回滚方案
- 发布时间

### 3. 执行重构

#### 重构原则

1. **小步迭代**：每次只做一个小改动
2. **频繁提交**：每个改动后提交
3. **测试保障**：确保测试通过
4. **行为不变**：重构不改变外部行为

#### 常见重构模式

| 重构 | 适用场景 | 示例 |
|------|----------|------|
| 提取函数 | 长函数 | `extractMethod()` |
| 重命名 | 命名不清 | `renameVariable()` |
| 移动函数 | 错位函数 | `moveMethod()` |
| 简化条件 | 复杂条件 | `simplifyConditional()` |
| 替换魔法值 | 硬编码常量 | `replaceMagicNumber()` |
| 合并重复 | 重复代码 | `consolidateDuplicate()` |

### 4. 验证和清理

#### 验证检查

- [ ] 所有测试通过
- [ ] 功能行为一致
- [ ] 性能没有退化
- [ ] 代码风格统一

#### 清理工作

- 删除死代码
- 更新注释
- 整理导入
- 格式化代码

## 重构原则

### SOLID 原则

| 原则 | 说明 | 应用 |
|------|------|------|
| **S**RP | 单一职责 | 每个类只做一件事 |
| **O**CP | 开闭原则 | 对扩展开放，对修改关闭 |
| **L**SP | 里氏替换 | 子类可替换父类 |
| **I**SP | 接口隔离 | 接口小而专注 |
| **D**IP | 依赖倒置 | 依赖抽象不依赖具体 |

### DRY 原则

**Don't Repeat Yourself**

- 识别重复模式
- 提取公共逻辑
- 创建可复用组件

### KISS 原则

**Keep It Simple, Stupid**

- 选择简单方案
- 避免过度设计
- 清晰优于巧妙

### YAGNI 原则

**You Aren't Gonna Need It**

- 不实现未来可能的功能
- 只解决当前问题
- 按需扩展

## 技术债务管理

### 债务分类

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| 无意债务 | 不知道更好的方式 | 学习后修复 |
| 有意债务 | 为速度妥协 | 计划偿还 |
| 疏忽债务 | 懒得做好 | 逐步改进 |
| 策略债务 | 业务决策 | 定期评估 |

### 债务追踪

```markdown
## 技术债务清单

### P0 - 紧急

- [ ] 债务 1：...
  - 影响：...
  - 估计工时：...
  - 计划偿还日期：...

### P1 - 重要

- [ ] 债务 2：...
```

### 偿还策略

- 每次开发偿还相关债务
- 固定时间（如 20%）用于还债
- 新代码零债务
- 定期债务审查

## 代码质量指标

### 度量标准

| 指标 | 目标 | 说明 |
|------|------|------|
| 函数长度 | <50 行 | 易于理解 |
| 类长度 | <500 行 | 单一职责 |
| 圈复杂度 | <10 | 易于测试 |
| 重复率 | <5% | 代码复用 |
| 测试覆盖率 | >80% | 质量保障 |

### 改进建议

```markdown
## 质量报告

### 当前状态

- 平均函数长度：35 行 ✅
- 平均类长度：280 行 ✅
- 圈复杂度：8.5 ✅
- 重复率：12% ⚠️
- 测试覆盖率：65% ⚠️

### 改进建议

1. 减少重复代码（12% → 5%）
   - 提取公共工具函数
   - 使用继承或组合

2. 提高测试覆盖率（65% → 80%）
   - 为核心功能添加测试
   - 添加边界测试用例
```
