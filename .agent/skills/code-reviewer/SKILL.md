# 代码审查员技能

> 代码质量、最佳实践和一致性检查的专家技能。

---

## 技能描述

**名称**: Code Reviewer  
**用途**: 在代码合并前进行质量检查、发现潜在问题、确保代码符合项目规范时使用此技能。

---

## 能力范围

### 核心能力

- 代码质量审查
- 最佳实践遵循检查
- 潜在 bug 识别
- 性能问题分析
- 安全性检查
- 代码风格一致性

---

## 审查流程

### 1. 准备阶段

- 理解变更目的
- 查看相关文件
- 了解影响范围

### 2. 代码审查

#### 功能正确性

- [ ] 代码实现了预期功能
- [ ] 边界情况已处理
- [ ] 错误处理完善
- [ ] 单元测试覆盖

#### 代码质量

- [ ] 遵循命名规范
- [ ] 函数简洁（<50 行）
- [ ] 无重复代码（DRY）
- [ ] 注释清晰有用

#### 性能

- [ ] 无不必要的计算
- [ ] 数据库查询优化
- [ ] 内存使用合理
- [ ] 避免阻塞操作

#### 安全性

- [ ] 输入验证完善
- [ ] 无敏感信息泄露
- [ ] 权限检查正确
- [ ] 遵循安全最佳实践

### 3. 问题分类

| 级别 | 说明 | 处理 |
|------|------|------|
| 🔴 严重 | 必须修复 | 阻塞合并 |
| 🟡 警告 | 建议修复 | 记录后合并 |
| 🟢 建议 | 可选优化 | 后续改进 |

### 4. 反馈格式

```markdown
## 审查结果

### 🔴 严重问题

1. **问题描述**
   - 文件：`xxx.swift`
   - 行号：123
   - 影响：可能导致崩溃
   - 建议：...

### 🟡 警告

1. **问题描述**
   - ...

### 🟢 建议

1. **优化建议**
   - ...

## 总体评价

代码质量：良好/中等/需要改进
建议：通过/修改后通过/重新审查
```

## 审查检查清单

### Swift 特定检查

- [ ] 使用 `async/await` 而非闭包
- [ ] 正确使用 `Result` 类型
- [ ] 避免循环引用（weak/unowned）
- [ ] 使用 `let` 而非 `var`（如可能）
- [ ] 遵循 SOLID 原则
- [ ] 使用 `guard` 进行早期返回
- [ ] 避免隐式解包可选值

### 架构检查

- [ ] 单一职责原则
- [ ] 依赖注入
- [ ] 清晰的层间边界
- [ ] 无紧耦合
- [ ] 可测试性

### 文档检查

- [ ] 公开 API 有文档注释
- [ ] 复杂逻辑有解释
- [ ] TODO/FIXME 已记录
- [ ] 变更记录已更新

## 常见问题模式

### 性能问题

```swift
// ❌ 低效
for item in items {
    if item.isValid {
        results.append(transform(item))
    }
}

// ✅ 高效
let results = items
    .filter { $0.isValid }
    .map { transform($0) }
```

### 内存问题

```swift
// ❌ 循环引用
buttonAction = {
    self.updateUI()  // self 强引用
}

// ✅ 正确
buttonAction = { [weak self] in
    self?.updateUI()
}
```

### 错误处理

```swift
// ❌ 忽略错误
try? fetchData()

// ✅ 正确处理
do {
    try await fetchData()
} catch {
    handleError(error)
}
```

## 审查原则

### ✅ 应该做的

- 对事不对人
- 提供具体建议
- 解释问题原因
- 给出代码示例
- 肯定做得好的地方

### ❌ 避免做的

- 人身攻击
- 模糊的批评
- 强加个人偏好
- 忽视上下文
- 过度吹毛求疵
