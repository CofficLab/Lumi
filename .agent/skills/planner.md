# 项目规划师技能

> 项目规划、任务分解和进度管理的专家技能。

---

## 技能描述

**名称**: Planner  
**用途**: 在项目启动、功能规划、任务分解和进度管理时使用此技能。

---

## 能力范围

### 核心能力

- 项目需求分析
- 任务分解和估算
- 风险识别和管理
- 进度跟踪
- 资源规划
- 优先级排序

---

## 规划流程

### 1. 需求分析

#### 收集信息

- 项目目标
- 用户需求
- 技术约束
- 时间限制
- 资源限制

#### 明确范围

| 包含 | 不包含 |
|------|--------|
| 功能列表 | 排除的功能 |
| 技术选型 | 暂不实现的功能 |
| 交付物 | 未来的扩展 |

### 2. 任务分解

#### WBS（工作分解结构）

```
项目
├── 阶段 1
│   ├── 任务 1.1
│   │   ├── 子任务 1.1.1
│   │   └── 子任务 1.1.2
│   └── 任务 1.2
├── 阶段 2
│   └── ...
```

#### 任务属性

每个任务应包含：

- 任务描述
- 预估工时
- 依赖关系
- 优先级
- 负责人

### 3. 时间估算

#### 估算方法

| 方法 | 用途 |
|------|------|
| 三点估算 | (乐观 +4× 最可能 + 悲观)/6 |
| 类比估算 | 参考类似任务 |
| 参数估算 | 基于历史数据 |
| 专家判断 | 团队讨论 |

#### 缓冲时间

- 任务级别：10-20%
- 阶段级别：15-25%
- 项目级别：20-30%

### 4. 风险评估

#### 风险识别

| 类别 | 示例 |
|------|------|
| 技术风险 | 新技术不熟悉 |
| 资源风险 | 人员变动 |
| 进度风险 | 依赖延迟 |
| 需求风险 | 需求变更 |

#### 风险矩阵

| 影响\概率 | 低 | 中 | 高 |
|----------|---|---|---|
| **高** | 中 | 高 | 极高 |
| **中** | 低 | 中 | 高 |
| **低** | 低 | 低 | 中 |

## 优先级排序

### MoSCoW 方法

| 优先级 | 说明 | 比例 |
|--------|------|------|
| **M**ust have | 必须有 | 60% |
| **S**hould have | 应该有 | 20% |
| **C**ould have | 可以有 | 15% |
| **W**on't have | 暂不实现 | 5% |

### Eisenhower 矩阵

| 重要\紧急 | 紧急 | 不紧急 |
|----------|------|--------|
| **重要** | 立即做 | 计划做 |
| **不重要** | 委托做 | 不做 |

## 使用示例

```
用户：为新功能制定开发计划
→ 分析需求，分解任务，估算时间，输出计划

用户：评估项目可行性
→ 分析风险，评估资源，给出建议

用户：跟踪项目进度
→ 更新状态，识别延迟，调整计划
```
